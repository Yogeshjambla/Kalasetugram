Database Fix Required

Your admin pages need some database columns to work properly

Issue: The admin pages are trying to access database columns that don't exist yet. This is normal for a new installation.

Quick Fix Steps

1
Run the Database Update Script

Click the button below to automatically add the missing database columns:

Fix Database Now
2
Alternative: Manual SQL Commands

If the automatic fix doesn't work, run these SQL commands in phpMyAdmin:

ALTER TABLE crafts ADD COLUMN featured BOOLEAN DEFAULT FALSE;
ALTER TABLE users ADD COLUMN status ENUM('active', 'inactive') DEFAULT 'active';
ALTER TABLE artisans ADD COLUMN specialization VARCHAR(200);
ALTER TABLE artisans ADD COLUMN location VARCHAR(200);
ALTER TABLE artisans ADD COLUMN verified BOOLEAN DEFAULT FALSE;
ALTER TABLE orders ADD COLUMN status ENUM('pending', 'confirmed', 'processing', 'shipped', 'delivered', 'cancelled') DEFAULT 'pending';
3
Access Your Admin Panel

After fixing the database, you can access all admin features:

Dashboard
Users
Crafts
Orders
Note: This is a one-time setup. After running the database update, all admin pages will work perfectly!
